# @double-bind/cli

## Purpose

Non-interactive command-line tool for database management, scripting, and Unix pipeline integration. Provides programmatic access to double-bind's functionality without a GUI.

## Commands

```bash
double-bind <command> [options]
```

| Command | Description |
|---------|-------------|
| `new <title>` | Create a new page |
| `daily` | Open or create today's daily note |
| `add <page> <content>` | Add a block to a page |
| `search <query>` | Full-text search |
| `query <cozoscript>` | Run a raw Datalog query |
| `list` | List all pages |
| `show <page>` | Display a page's blocks |
| `import <format> <file>` | Import from file (roam-json, markdown) |
| `export <format> [output]` | Export to file (markdown, json) |
| `backup <path>` | Create database backup |
| `restore <path>` | Restore from backup |
| `stats` | Show database statistics |
| `init [path]` | Initialize a new database |

### Output Formats

```bash
# Default: human-readable
double-bind list
# My Page                    2025-03-15
# Meeting Notes               2025-03-14

# JSON output for scripting
double-bind list --json
# [{"pageId":"...","title":"My Page","createdAt":...}, ...]

# TSV output for Unix pipes
double-bind list --tsv
# page_id\ttitle\tcreated_at
# ...\tMy Page\t2025-03-15
```

### Unix Pipe Examples

```bash
# Find orphan pages
double-bind query '?[title] := *pages{ page_id, title }, not *links{ target_id: page_id }' --tsv

# Count blocks per page
double-bind query '?[title, count(block_id)] := *blocks{ block_id, page_id }, *pages{ page_id, title }' --tsv | sort -t$'\t' -k2 -rn

# Export specific pages to markdown
double-bind list --json | jq -r '.[].pageId' | head -5 | xargs double-bind export markdown

# Fuzzy find and open
double-bind list --titles-only | fzf | xargs double-bind show
```

## Dependencies

- `@double-bind/core` — business logic
- `cozo-node` — direct CozoDB access (no Tauri)
- Argument parser (minimist, commander, or yargs)

## Database Connection

The CLI connects directly to CozoDB via `cozo-node`, not via Tauri:

```typescript
import { CozoDb } from 'cozo-node';

const dbPath = process.env.DOUBLE_BIND_DB_PATH || getDefaultDbPath();
const db = new CozoDb('rocksdb', dbPath);
```

The CLI and desktop app share the same database file. Only one should access it at a time (RocksDB handles file locking).

## Resolved Decisions

- **Argument parser**: `commander` — mature, zero-config subcommand support, built-in help generation. Chosen over yargs (heavier) and minimist (too minimal for subcommands).
- **Shell completions**: Generated by commander's built-in completion support. Bash/zsh/fish scripts emitted via `double-bind completions <shell>`.
- **Exit codes**: `0` = success, `1` = general error, `2` = usage error (bad arguments), `3` = database error, `4` = file I/O error. Consistent across all commands.
- **Verbosity flags**: `--verbose` (`-v`) enables debug logging to stderr. `--quiet` (`-q`) suppresses all non-error output. Default: normal output to stdout, errors to stderr.
- **Database path resolution**: (1) `--db <path>` flag, (2) `DOUBLE_BIND_DB_PATH` env var, (3) `~/.config/double-bind/config.json` → `dbPath`, (4) platform default (same paths as desktop app). First match wins.
